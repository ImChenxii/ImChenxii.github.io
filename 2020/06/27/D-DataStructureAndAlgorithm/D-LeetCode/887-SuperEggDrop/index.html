LeetCode 题解系列(887)：鸡蛋掉落

题目描述:

你将获得 K 个鸡蛋，并可以使用一栋从 1 到 N  共有 N 层楼的建筑。

每个蛋的功能都是一样的，如果一个蛋碎了，你就不能再把它掉下去。

你知道存在楼层 F ，满足 0 <= F <= N 任何从高于 F 的楼层落下的鸡蛋都会碎，从 F 楼层或比它低的楼层落下的鸡蛋都不会破。

每次移动，你可以取一个鸡蛋（如果你有完整的鸡蛋）并把它从任一楼层 X 扔下（满足 1 <= X <= N）。

你的目标是确切地知道 F 的值是多少。

无论 F 的初始值如何，你确定 F 的值的最小移动次数是多少？

 

示例 1：

输入：K = 1, N = 2
输出：2
解释：
鸡蛋从 1 楼掉落。如果它碎了，我们肯定知道 F = 0 。
否则，鸡蛋从 2 楼掉落。如果它碎了，我们肯定知道 F = 1 。
如果它没碎，那么我们肯定知道 F = 2 。
因此，在最坏的情况下我们需要移动 2 次以确定 F 是多少。
示例 2：

输入：K = 2, N = 6
输出：3
示例 3：

输入：K = 3, N = 14
输出：4  

提示：

1 <= K <= 100
1 <= N <= 10000

解法一: 动态规划

  1. 定义状态：当前拥有的鸡蛋数 K 和需要测试的楼层数 N。随着测试的进行，鸡蛋个数可能减少，楼层的搜索范围会减小，这就是状态的变化。dp[i][j]：一共有 i 层楼梯（注意：这里 i 不表示高度）的情况下，使用 j 个鸡蛋的最少实验的次数。说明：i 表示的是楼层的大小，不是高度（第几层）的意思，例如楼层区间 [8, 9, 10] 的大小为 33。j 表示可以使用的鸡蛋的个数，它是约束条件。
  2. 选择：其实就是去选择哪层楼扔鸡蛋。在某个层数中，有多个选择，选择每个层数之后都有两种情况：碎或不碎，而至少就是指的在两种情况中选择最差的情况，但至少又体现在在本次所有可能的选择中，选择一个使得最坏情况最少的选择。所以大体的转移方程就能列出：dp[i][j] 就是对于 1~i 每个取值选择一个使得 max(鸡蛋碎，鸡蛋没碎)最小的 i 对应的值。
  3. 选择在第 i 层楼扔鸡蛋后可能有两种情况：鸡蛋碎了，鸡蛋没碎。如果鸡蛋碎了，那么鸡蛋的个数 K 应该减一，搜索的楼层区间应该从 [1..N] 变为 [1..i-1] 共 i-1 层楼；如果鸡蛋没碎，那么鸡蛋的个数 K 不变，搜索的楼层区间应该从 [1..N] 变为 [i+1..N] 共 N-i 层楼。因为要求的是最坏情况下扔鸡蛋的次数，所以鸡蛋在第 i 层楼碎没碎，取决于那种情况的结果更大。所以得到一般位置具体的状态转移方程：dp[i][j]=min(k=1~i)(max(dp[k-1][j-1],dp[i-k],j)+1)。
  4. base case：第 0 行，楼层为 0 的时候，不管鸡蛋个数多少，都测试不出鸡蛋的个数，故全为 0；楼层为 1 的时候，0 个鸡蛋的时候，扔 0 次，1 个以及 1 个鸡蛋以上只需要扔 1 次；第 0 列，鸡蛋个数为 0 的时候，不管楼层为多少，也测试不出需要鸡蛋的个数，为了方便后面计算，设为 0；第 1 列，鸡蛋个数为 1 的时候，最少次数就是楼层的高度，因为只能从第 1 层一层一层往上试，最差就是顶层才碎。
  5. 结果：dp 矩阵右下角的值。
  6. 优化：二分查找楼层，可以观察转移方程发现，最外层的变量是k，枚举了扔下鸡蛋的楼层的高度，这里它是自变量，将其余的 i 和 j 视为常数，dp[k - 1][j - 1]：根据语义，k 增大的时候，楼层大小越大，它的值就越大；dp[i - k][j]：根据语义，k 增大的时候，楼层大小越小，它的值就越小。可以得出dp[k - 1][j - 1]是单调不减的，dp[i - k][j]是单调不增的，并且它们的值都是整数。二者的较大值的最小点在它们交汇的地方，也有可能不交汇，二者较大值的最小者一定出现在画成曲线段交点的两侧，并且二者的差值不会超过 1，也就是如果没有重合的点，两边的最大值是一样的，因此取左侧和右侧两点中的一点都可以，不失一般性，可以取左边的那个点的 k。也就是找到使得 dp[i - k][j] <= dp[k - i][j - 1] 最大的那个 k 值即可。这里使用二分查找算法。关键在于 dp[i - k][j] > dp[k - i][j - 1] 的时候，k 一定不是要找的，根据这一点写出二分的代码。

时间复杂度: O(NKlogN); 辅助空间复杂度: O(NK)

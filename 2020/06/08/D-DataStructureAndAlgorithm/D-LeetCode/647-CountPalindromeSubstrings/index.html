LeetCode 题解系列(647)： 回文字串

题目描述:

给定一个字符串，你的任务是计算这个字符串中有多少个回文子串。

具有不同开始位置或结束位置的子串，即使是由相同的字符组成，也会被计为是不同的子串。

示例 1:

输入: “abc”
输出: 3
解释: 三个回文子串: “a”, “b”, “c”.
示例 2:

输入: “aaa”
输出: 6
说明: 6个回文子串: “a”, “a”, “a”, “aa”, “aa”, “aaa”.

解法一: 中心扩展法

  1. 在长度为 N 的字符串中，可能的回文串中心位置有 2N-1 个：字母或两个字母中间。
  2. 当 center 变量是偶数时，代表是字母，如果是奇数时，代表是字母中间
  3. 此时就能确定中心左边位置：left = center / 2，如果是字母就是 center 位置，如果是字母中间，就是左边位置字母；确定右边第一个位置right = left + center % 2，如果是字母就是 center，不是就是 left 右一个位置。
  4. 对于每一个可能的 center，开始以其为中心向两边扩，每扩成功一次，以当前位置为中心的回文子串数量+1

时间复杂度: O(N^2); 辅助空间复杂度: O(1)

解法二：Manacher 算法

因为 Manacher 算法能够得到每个位置作为中心的回文半径，也就能统计出子数组个数：

  1. 将首尾使用两个不同的特殊字符填充且不能是#，来做一个回文截断
  2. 构建一个辅助数组，就是 Manacher 算法使用的#填充的辅助数组
  3. 定义变量 pR 代表当前回文半径能到达的最右位置，定义变量 index，代表对应 pR 的中心位置
  4. 开始遍历辅助数组的每一个位置 i，如果当前 pR 能包住 i，找到 i 关于 index 的对称位置的回文半径，因为 pR 只能保证 i 到 pR 之间的回文性，所以赋值 pR-i 和对称位置回文半径较小者
  5. 因为可能没包住，也可能包住了 pR 只保证了 i 到 pR 的回文半径有效，所以可能继续扩
  6. 扩完之后，可能 i 位置的回文半径对应位置超过了 pR，到达了更右的位置，所以更新 index 为 i，更新更右位置为 i+当前回文半径
  7. 统计完所有回文半径后，开始统计子数组的个数，对于辅助数组每个位置相当于一个中心，因为有#填充，所以半径+1/2是当前中心的回文字串个数

时间复杂度: O(N); 辅助空间复杂度: O(N)

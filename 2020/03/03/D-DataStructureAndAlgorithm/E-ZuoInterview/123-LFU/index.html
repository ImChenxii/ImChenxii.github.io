代码面试指南系列（一百二十三）：LFU缓存设计

题目描述:

一个缓存结构需要实现如下功能：

  1. void set(int key, int value): 加入或修改key对应的value
  2. int get(int key): 查询key对应的value值

但是缓存中最多放K条记录，如果新的第K+1条记录要加入，就需要根据策略删掉一条记录，然后才能把新纪录加入。这个策略为：在缓存结构的K条记录中，哪一个key从进入缓存结构的时刻开始，被调用set或get的次数最小，就删掉这个key的记录；如果调用次数最少的key有多个，上次调用发生最早的key被删除。

要求：set和get方法的时间复杂度为O(1)

解法:

整体结构设计成发生set和get操作次数一样的key，都放在一个双向链表里(桶)。对于不同的操作次数，分别设置桶，然后桶和桶之间按照操作次数从小到大串起来，同合同之间也看做是一个双向链表。
当某一个key发生set和get时，来到key所在的位置，把key从原来的桶中去掉，也就是key的上一个节点和下一个节点之间直接连接，然后把key扔到次数+1之后的桶中，如果没有次数+1的桶就新建桶，保证桶之间依然是双向链表的链接；如果已经有次数+1的桶，就把key放在这个桶的头部。如果key原来所在的桶中只有key这一个记录，就删掉原来的桶，保证桶之间依然是双向链表的链接。

LeetCode 题解系列(355)：设计推特

题目描述:

设计一个简化版的推特(Twitter)，可以让用户实现发送推文，关注/取消关注其他用户，能够看见关注人（包括自己）的最近十条推文。你的设计需要支持以下的几个功能：

postTweet(userId, tweetId): 创建一条新的推文
getNewsFeed(userId): 检索最近的十条推文。每个推文都必须是由此用户关注的人或者是用户自己发出的。推文必须按照时间顺序由最近的开始排序。
follow(followerId, followeeId): 关注一个用户
unfollow(followerId, followeeId): 取消关注一个用户
示例:

Twitter twitter = new Twitter();

// 用户1发送了一条新推文 (用户id = 1, 推文id = 5).
twitter.postTweet(1, 5);

// 用户1的获取推文应当返回一个列表，其中包含一个id为5的推文.
twitter.getNewsFeed(1);

// 用户1关注了用户2.
twitter.follow(1, 2);

// 用户2发送了一个新推文 (推文id = 6).
twitter.postTweet(2, 6);

// 用户1的获取推文应当返回一个列表，其中包含两个推文，id分别为 -> [6, 5].
// 推文id6应当在推文id5之前，因为它是在5之后发送的.
twitter.getNewsFeed(1);

// 用户1取消关注了用户2.
twitter.unfollow(1, 2);

// 用户1的获取推文应当返回一个列表，其中包含一个id为5的推文.
// 因为用户1已经不再关注用户2.
twitter.getNewsFeed(1);

解法一: 大顶堆+链表

  1. 推特数据结构主要包括四部分数据结构：1.用户 id 到对应推特链表的映射 map，key 是 id，value 是推文单链表；2.用户对应的关注 map，key 是用户 id，value 是一个 set，存储关注的用户。3.时间戳，是一个全局整型变量，每次发推文时，都会更新并赋值给当前推文。4.存储单链表的大顶堆，排序是根据每个推文对应链表头部的时间戳大小。
  2. 当发送一条推文时，时间戳+1，如果该用户之前发布过推文，将推文加到对应用户推文单链表的头部，如果没有发布过，那么直接建立新的推文链表结构，并添加到头部。
  3. 当关注和取消关注用户时，在对应的 set 中添加或移除即可。
  4. 当获取最新的十条推文时，先将自身和关注人的所有推特链表入堆，并开始出堆，因为堆顶一定是时间戳最大的推文。每次移除一个元素，还要将下一条推文入堆。

时间复杂度: O(N); 辅助空间复杂度: O(N)

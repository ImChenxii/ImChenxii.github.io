LeetCode 题解系列(440)：字典序的第K小数字

题目描述:

给定整数 n 和 k，找到 1 到 n 中字典序第 k 小的数字。

注意：1 ≤ k ≤ n ≤ 109。

示例 :

输入:
n: 13 k: 2

输出:
10

解释:
字典序的排列是 [1, 10, 11, 12, 13, 2, 3, 4, 5, 6, 7, 8, 9]，所以第二小的数字是 10。

解法一: 数学归纳

  1. 可以看做每个节点都拥有 10 个孩子节点，因为作为一个前缀 ，它后面可以接 0~9 这十个数字。而且可以发现，整个字典序排列也就是对十叉树进行先序遍历。
  2. 确定指定前缀下所有子节点数：用下一个前缀的起点减去当前前缀的起点，那么就是当前前缀下的所有子节点数总和，但是应该注意下一个前缀是否超过 n 了，超过 n 代表下一个前缀取不到那么大，应该取到的是 n+1，因为本身有 n 这个位置，直接相减差一个数字，所以是 n+1。
  3. 如果第k个数在当前前缀下，那么prefix *= 10，到下一级去具体判断；如果第k个数不在当前前缀下：前缀+1，到下一级前缀去判断。

时间复杂度: O(N); 辅助空间复杂度: O(N)

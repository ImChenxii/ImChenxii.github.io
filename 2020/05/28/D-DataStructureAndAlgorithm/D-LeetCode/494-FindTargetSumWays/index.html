LeetCode 题解系列(494)：目标和

题目描述:

给定一个非负整数数组，a1, a2, …, an, 和一个目标数，S。现在你有两个符号 + 和 -。对于数组中的任意一个整数，你都可以从 + 或 -中选择一个符号添加在前面。

返回可以使最终数组和为目标数 S 的所有添加符号的方法数。

示例 1:

输入: nums: [1, 1, 1, 1, 1], S: 3
输出: 5
解释:

-1+1+1+1+1 = 3
+1-1+1+1+1 = 3
+1+1-1+1+1 = 3
+1+1+1-1+1 = 3
+1+1+1+1-1 = 3

一共有5种方法让最终目标和为3。
注意:

数组非空，且长度不会超过20。
初始的数组的和不会超过1000。
保证返回的最终结果能被32位整数存下。

解法一: 递归

  1. 递归函数的意义是返回从当前索引和累加和开始，能否组成累加和 S。
  2. 递归函数的终止条件就是到了数组的末尾，此时就要判断当前的累加和是否等于目标 S，如果等于，返回 1，否则返回 0。
  3. 往下递归时，每次都有两个选择，分别更新当前的累加和，并进入下一层递归。

时间复杂度: O(2^N); 辅助空间复杂度: O(1)

解法二：动态规划

  1. 从递归函数可以看出，递归的转移条件就是当前的累加和和当前的位置，所以可以用这两者作为动态规划的轴。
  2. 因为数组和不会超过 1000，所以范围是-1000~1000，dp[i][j]代表的就是 nums[0..i] 能够组成 j 的种数。因为索引不能为负数，所有有一个 1000 的偏移量。
  3. base case：对于只有 nums[0] 时，只能够组成 dp[0][nums[0]+1000]=1 和 dp[-nums[0]+1000]=1，其余位置都为 0。
  4. 一般位置：当前位置可能是加了，也可能是减了，所以当前位置(i,j) 可能是由(i-1,j+nums[i]) 构成或者(i-1,j-nums[i]) 构成，两种情况相加，就是构成当前和的种数。
  5. 结果：dp[len-1][S+1000]，1000 是偏移量。

时间复杂度：O(2000N); 辅助空间复杂度：O(2000N)

LeetCode 题解系列(279)：完全平方数

题目描述:

给定正整数 n，找到若干个完全平方数（比如 1, 4, 9, 16, …）使得它们的和等于 n。你需要让组成和的完全平方数的个数最少。

示例 1:

输入: n = 12
输出: 3
解释: 12 = 4 + 4 + 4.

示例 2:

输入: n = 13
输出: 2
解释: 13 = 4 + 9.

解法一: 动态规划

  1. 建立动态规划矩阵，dp[i] 代表组成 i 所需要的最少的完全平方数数量。
  2. base case：dp[0]=0，代表组成 0 不需要完全平方数，是为了方便求解后面的一般位置。
  3. 一般位置：对于当前 i 所有可能的完全平方数，即最小是 1，最大是 sqrt(i)，全部遍历，并减去这个完全平方数 n，即 i-n 后会得到一个较少的数字，而这个数字的完全平方数个数之前已经算出来过为 x，x+1 就是当前的数量，对于全部可能的 x+1 取最小的一个，就是当前位置能够组成 i 的完全平方数最少的数量。
  4. 初始时，对每个位置先填充为 n，最多就是 n 个 1 组成。

时间复杂度: O(N); 辅助空间复杂度: O(N)

解法二: 贪心+递归

  1. 一个数字 n，最少的组成个数就是 1，即 n 本身就是完全平方数；最多的组成个数就是 n，即由 n 个 1 组成。
  2. 对于所有的可能，从 1 遍历到 n，找到最少的就停止返回。
  3. 写一个递归函数，函数的作用就是判断当前的 n 能否由 count 个完全平方数组成，当前仅当 count 为 1，n 是一个完全平方数时，递归才会终止；否则，对所有可能减去的完全平方数后，count-1 后，再去递归判断。

时间复杂度: O(N); 辅助空间复杂度: O(N)

解法三: 数学法

  1. 根据四平方和三平方定理，如果一个数字是否是4^k x (8m+7)，如果是，必定是四个平方和组成的，不是的话，可能是 3、2、1 个组成的。
  2. 如果 n 对 4 取余为 0，代表还可以除 4，不断除 4，直到不是 4 的倍数时，判断是否是 8m+7，即对 8 取余判断结果是否为 7，如果为 7，那么直接返回 4，如果不是，可能就是 1、2、3，继续后面的判断。
  3. 判断是否为 1，就判断是否是一个完全平方数，即对 sqrt(n) 取整然后平方判断是否等于 n；对于是否为 2，先枚举一个平方数，再判断差值是否是平方数，如果有满足的组合，返回 2；如果 1 和 2 都不是，那么就是 3，返回 3。

时间复杂度: O(N); 辅助空间复杂度: O(1)

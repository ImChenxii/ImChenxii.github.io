LeetCode 题解系列(403)：青蛙过河

题目描述:

一只青蛙想要过河。 假定河流被等分为 x 个单元格，并且在每一个单元格内都有可能放有一石子（也有可能没有）。 青蛙可以跳上石头，但是不可以跳入水中。

给定石子的位置列表（用单元格序号升序表示）， 请判定青蛙能否成功过河（即能否在最后一步跳至最后一个石子上）。 开始时， 青蛙默认已站在第一个石子上，并可以假定它第一步只能跳跃一个单位（即只能从单元格1跳至单元格2）。

如果青蛙上一步跳跃了 k 个单位，那么它接下来的跳跃距离只能选择为 k - 1、k 或 k + 1个单位。 另请注意，青蛙只能向前方（终点的方向）跳跃。

请注意：

石子的数量 ≥ 2 且 < 1100；
每一个石子的位置序号都是一个非负整数，且其 < 231；
第一个石子的位置永远是0。
示例 1:

[0,1,3,5,6,8,12,17]

总共有8个石子。
第一个石子处于序号为0的单元格的位置, 第二个石子处于序号为1的单元格的位置,
第三个石子在序号为3的单元格的位置， 以此定义整个数组…
最后一个石子处于序号为17的单元格的位置。

返回 true。即青蛙可以成功过河，按照如下方案跳跃：
跳1个单位到第2块石子, 然后跳2个单位到第3块石子, 接着
跳2个单位到第4块石子, 然后跳3个单位到第6块石子,
跳4个单位到第7块石子, 最后，跳5个单位到第8个石子（即最后一块石子）。
示例 2:

[0,1,2,3,4,8,9,11]

返回 false。青蛙没有办法过河。
这是因为第5和第6个石子之间的间距太大，没有可选的方案供青蛙跳跃过去。

解法一: 递归

  1. 整体过程基于 DFS，每次都根据上一次跳的步长达到的位置首先判断是否有这块石头来判断是否能继续往后走，然后再根据上次跳的步长来跳到三个位置，然后下一次迭代函数中去判断是否有石头。
  2. 递归的终止就是到达了目的地石头，整体过程需要当前跳的位置和跳到当前位置的步长，来进行下一次递归。

时间复杂度: O(N^3); 辅助空间复杂度: O(N)

解法二：动态规划

  1. 首先建立一个 HashMap，map 的 key 是当前的石头，map 的 value 是能跳到当前这块石头可能的所有步长。
  2. 初始时把所有石头值添加进 map，并给每个 map 初始化一个 HashSet。
  3. 然后给 0 位置石头添加步长 0 开始跳跃。
  4. 然后从第 0 个石头开始遍历，根据第 i 个石头每个所有可能跳过来的步长，能确定下一步跳的三个步长，并判断这三个步长加上当前石头后有没有目标石头，如果有的话，把步长添加到这个目标石头中。因为整体是从前向后的，所以遍历到每个石头的时候跳到这个石头所有的步长是完备没有缺失的(因为只能从前面的跳到当前石头上)。
  5. 最后判断最后一个石头上有没有能够跳过来的步长，有的话就代表可以跳到，没有的话就代表失败。

时间复杂度：O(N)；辅助空间复杂度：O(N)

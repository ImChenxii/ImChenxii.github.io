LeetCode 题解系列(315)：计算右侧小于当前元素的个数

题目描述:

给定一个整数数组 nums，按要求返回一个新数组 counts。数组 counts 有该性质： counts[i] 的值是  nums[i] 右侧小于 nums[i] 的元素的数量。

示例:

输入: [5,2,6,1]
输出: [2,1,1,0]
解释:
5 的右侧有 2 个更小的元素 (2 和 1).
2 的右侧仅有 1 个更小的元素 (1).
6 的右侧有 1 个更小的元素 (1).
1 的右侧有 0 个更小的元素.

解法一: 归并排序+索引数组

  1. 因为要记录每个元素的右侧更小元素个数，所以不能直接使用原数组进行归并，因为归并的时候无法根据元素值还原其对应索引，但是可以根据索引查询到其对应值，所以使用索引数组来进行归并排序。
  2. 建立 count 数组，代表统计 nums 每个位置数字右边比其小的元素的个数；建立 indexes，存储的是排序的索引列表；建立 tmp 数组，用来辅助索引排序。
  3. 归并排序仍然是先递归分割，再合并。合并时，以左边元素添加进结果数组时统计为基准，如果左边 l 位置入排序好的数组时，右边比其小的元素个数就是 r-mid-1，即右边已经入排序数组的元素个数；如果 r>mid 了，说明此时 l 和 l 到 mid-1 之间的元素，都有 r-mid 个元素比其小。
  4. 这些统计和合并都是在索引上发生的。

时间复杂度: O(NlogN); 辅助空间复杂度: O(N)

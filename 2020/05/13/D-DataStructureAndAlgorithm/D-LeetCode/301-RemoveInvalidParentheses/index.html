LeetCode 题解系列(301)：删除无效括号

题目描述:

删除最小数量的无效括号，使得输入的字符串有效，返回所有可能的结果。

说明: 输入可能包含了除 ( 和 ) 以外的字符。

示例 1:

输入: “()())()”
输出: [“()()()”, “(())()”]
示例 2:

输入: “(a)())()”
输出: [“(a)()()”, “(a())()”]
示例 3:

输入: “)(“
输出: [“”]

解法一: DFS

本质上还是组合问题，使用当前有效的字符串和无效的长度去递归，满足条件了就添加。

  1. 先统计出无效的左括号和右括号个数，右括号无效是因为左边没有左括号和其匹配，左括号无效是因为多出来
  2. 根据统计出的左右括号的无效数量，递归过程中的关键就是遇到左右括号时开始产生递归分支：即是否选择当前括号，如果选择，当前临时字符串括号数量+1，如果不选择，对应的括号无效数量-1
  3. 递归的终止条件就是遍历到了最后一个位置，并且此时左右无效括号数量都为 0。
  4. 递归过程有一个剪枝，就是当前的左括号数量如果不大于右括号数量时，是无法进入当前右括号这个递归的，代表只能选择删除当前右括号。

时间复杂度: O(2^N); 辅助空间复杂度: O(N)

解法二：BFS

BFS 也是和无效的括号数量有关，仍然是尝试删除。

  1. 将完整的字符串先添加进队列，并统计当前无效数量cnt，cnt>0代表存在无效左括号，cnt<0 代表存在无效右括号，左括号无法确定。
  2. 此时如果 cnt>0，就对当前元素所有的左括号数量-1 后的字符串添加进队列，此时要进行一个判重操作，防止重复；如果 cnt<0，就把所有可能的右括号位置删除一个都添加进队列。
  3. 添加完成后，重新判断，根据当前 cnt 继续选择删除策略。
  4. 如果 cnt 为 0 了，代表有效了，添加当前满足的字符串
